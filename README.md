# PolygonDetectorC

Необходимо реализовать gRPC сервер на С, который принимает от клиента запрос с изображением и набором полигонов произвольной формы (каждый полигон состоит из произвольного количества точек >3), после чего детектирует объекты внутри заданных полигонов (объект считается принадлежащим полигону, если площадь пересечения бокса объекта с полигоном составляет не менее заданного порогового значения, задаваемое индивидуально для каждого полигона). Каждый полигон имеет тип (включающий или исключающий), приоритет обработки, пороговое значение для включения и список классов объектов для детекции. Сервер должен детектировать объекты нескольких различных классов (хотя бы 5). Можно взять любую готовую модель, например, YOLO, и готовые веса, обученные на COCO, используя dnn модуль из OpenCV для инференса. При пересечении полигонов применяется правило обработки согласно приоритетам: полигон с более высоким приоритетом переопределяет правила полигонов с меньшим приоритетом в зоне пересечения. Сервер возвращает изображение с отрисованными задетектированными объектами, учитывая логику включающих/исключающих зон и фильтры классов. Также необходимо написать клиента, который отправляет изображение и конфигурацию полигонов на сервер и выводит на экран результат обработки (imshow). Конфигурация полигонов должна поддерживать гибкое задание правил для различных сценариев использования. Необходимо обернуть решение в Docker и задание выложить на GitHub/GitLab

Требования:
1. Оформить код согласно Google Style Guide (отступы, названия переменных/классов/функций и т.д)
2. Названия коммитов оформить согласно Conventional commits
3. Организовать сборку с помощью CMake
4. Задокументировать основные классы/функции Doxygen
5. Написать unit тесты.
6. Написать readme с инструкцией по сборке и тестированию
7. Геометрические примитивы и все операции над ними реализовать без использования сторонних библиотек (Boost.Geometry и т.д)

Что будет проверяться:

- Корректное управление памятью

- Архитектурные решения

- Соответствие указанным требованиям
